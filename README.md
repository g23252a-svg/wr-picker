# WR Picker v5.0

와일드 리프트 드래프트에서 라인, 아군, 상대, 밴을 입력하면 **매일 자동 갱신되는** 역할별 통계와 패치 추세, 팀 조합, 상성, 개인 숙련도를 함께 계산하는 설치형 웹 앱입니다.

## v5.0 핵심 개선 (자동 데이터 파이프라인)

- **스탯 자동 갱신**: GitHub Actions가 매일 WildRiftFire 통계를 받아 커밋합니다. 수동 갱신이 더 이상 필요하지 않습니다
- **런타임 데이터 로딩**: 앱이 `data/latest.json`을 우선 사용하고, 실패하면 캐시 → 번들 스냅샷 순으로 폴백합니다. 앱을 다시 배포하지 않아도 통계가 최신으로 유지됩니다
- **랭크 구간 3종 수집**: Diamond+ / Master+ / Challenger 전체를 저장합니다 (UI 선택기는 다음 패치)
- **일자별 아카이브**: `data/history/<날짜>.json`이 쌓이며 추세 계산의 기준선이 됩니다
- 업스트림의 주 라인 플래그(`main`)와 공식 티어(`tier`)를 함께 수집해 주 라인 판정을 추정이 아닌 실제 값으로 대체
- 서비스 워커가 통계 경로만 네트워크 우선으로 처리해 캐시된 옛 데이터가 고정되는 문제 제거
- 스모크 테스트를 특정 수치가 아닌 불변식 검증으로 전환해 매일 갱신돼도 깨지지 않음

## v4.2 개선 (패치 추세 엔진)

- 직전 스냅샷을 함께 내장해 역할별 승률 **추세(▲/▼ %p)** 를 계산·표시
- 추세를 표본 신뢰도로 감쇠해 메타 축에 소폭 반영(±2.5점 한도) — `패치 상승세`/`패치 하락세` 배지
- 추천 카드·검색 드롭다운·지금 픽 요약에 추세 정보 노출
- 패치 라벨·데이터 상태줄이 스냅샷 메타를 그대로 따라가도록 동적화

## v4.1 개선

- 픽률 기반 표본 신뢰도로 승률을 평균 쪽으로 축소해 저표본 과대평가 완화
- 밴률은 실력 점수가 아닌 가용성 위험으로 분리 표시
- `지금 픽`, 안전 대안, 카운터 대안을 추천 목록 위에서 즉시 확인
- 스냅샷 경과일에 따라 데이터 상태와 추천 신뢰도를 자동 조정

- 승률·픽률·밴률 역할 행을 챔피언/포지션별로 분리
- 저픽률 고승률을 평균으로 축소하는 표본 신뢰도 보정
- 조합·카운터·메타·숙련 점수를 0–100 가중평균으로 제공
- 역할 통계가 있는 비주류 포지션까지 자동 후보에 포함
- 드래프트 자동 저장, 40단계 실행취소/다시실행, URL 공유
- 다크/라이트 테마와 모바일·태블릿·데스크톱 반응형 2열 UI
- 키보드 검색 탐색, 포커스 트랩, 스크린리더 상태 안내, 확대 허용
- JSON 파일 기반 전적 백업/복원과 안전한 데이터 마이그레이션
- 설치 프롬프트, 오프라인 앱 셸, 새 버전 업데이트 알림

## 실행

별도 빌드 없이 정적 서버로 실행합니다.

```bash
python -m http.server 4173
```

브라우저에서 `http://localhost:4173`을 엽니다. `file://`로도 핵심 기능은 동작하지만 서비스 워커와 설치 기능은 HTTP(S)에서만 사용할 수 있습니다.

## 검증

Node.js 18 이상에서 외부 패키지 없이 실행됩니다.

```bash
npm test
```

검증 항목은 앱 스크립트 문법, 챔피언 141종과 안정 ID, 역할 통계 불변식, 번들 스냅샷과 `data/latest.json`의 정합성, manifest, 서비스 워커 캐시 전략, 기본 접근성 규칙입니다.

## 데이터 파이프라인

```bash
node scripts/fetch-stats.mjs
```

WildRiftFire 통계 페이지에 인라인된 JSON 페이로드를 받아 세 곳에 씁니다.

| 산출물 | 용도 |
| --- | --- |
| `data/latest.json` | 앱이 실행 시 받아오는 최신 통계 (랭크 구간 3종 + 추세 기준선) |
| `data/history/<날짜>.json` | 업스트림 갱신일 기준 아카이브, 추세 기준선의 출처 |
| `stats.js` | 앱 셸에 번들되는 오프라인 폴백 (Diamond+) |

`--from <파일>`로 저장된 페이지를 파싱하거나 `--dry-run`으로 쓰기 없이 점검할 수 있습니다. 챔피언 DB에 없는 신규 챔피언이 나오면 해당 행만 건너뛰고 종료 코드 2와 함께 이름을 출력하며, CI는 이를 경고로 표시하되 나머지 갱신은 계속 진행합니다.

`.github/workflows/update-stats.yml`이 매일 04:20 UTC(13:20 KST)에 이 스크립트를 실행하고, 스모크 테스트를 통과하며 실제 변경이 있을 때만 커밋합니다. Actions 탭에서 수동 실행도 가능합니다.

## 데이터 메모

통계 출처는 중국 서버(lolm.qq.com) 랭크 데이터이며, 티어 등급은 WildRiftFire 티어 리스트를 따릅니다. 앱 화면에 패치·지역·랭크 구간·스냅샷 날짜를 표시하고, 통계가 없는 챔피언/역할은 메타 축을 중립으로 처리합니다. 추세는 양쪽 스냅샷 픽률이 1% 이상일 때만 계산합니다.

`stats.js`와 `data/`는 생성 파일이므로 직접 수정하지 말고 파이프라인을 다시 실행하세요.
